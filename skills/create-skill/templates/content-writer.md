# Content Writer Template

Use this template when building a skill for writing in a consistent voice — blog posts, social media, newsletters, email copy, or any recurring writing task.

## How to Customize

Replace everything in `[brackets]` with the user's specifics. Delete sections that don't apply.

## Content Structure

### [Content Type 1, e.g., "Blog Posts"]
- **Length:** [e.g., 800-1200 words]
- **Structure:** [e.g., hook → context → main points → takeaway]
- **Opening style:** [e.g., start with a question, start with a bold statement, start with a story]

### [Content Type 2, e.g., "Social Posts"]

## Topics and Themes

The core topics [name] writes about:
1. [Topic 1 — brief description]
2. [Topic 2 — brief description]
3. [Topic 3 — brief description]

These topics often connect. When possible, tie content back to [overarching theme or mission].

## Rules

### Always
- [e.g., Include a clear takeaway or actionable insight]
- [e.g., Back up claims with specific examples or data]
- [e.g., End with something that invites engagement]

### Never
- [e.g., Use clickbait or sensational headlines]
- [e.g., Write generic advice that could come from anyone]
- [e.g., Use corporate buzzwords like "synergy", "leverage", "optimize"]

## Process

1. **Understand the brief.** Ask what the piece is about, who it's for, and where it will be published. If any of these are unclear, ask before writing.
2. **Draft the content** following the voice, structure, and rules above.
3. **Self-review.** Before presenting the draft, check: Does this sound like [name]? Is the structure right for the platform? Is there a clear takeaway? Would [name] actually post this?
4. **Present the draft** and ask if anything needs adjusting.

## Customization Checklist

When filling in this template, make sure you've captured:

- [ ] 2-3 real voice examples (what they sound like)
- [ ] 1-2 anti-examples (what they don't sound like)
- [ ] Specific content formats with length and structure
- [ ] Core topics and themes
- [ ] Hard rules (always/never)
- [ ] Platform-specific details (if applicable)
